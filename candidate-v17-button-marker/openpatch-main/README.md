# OpenPatch

OpenPatch is a Chrome extension that sends AI-generated code packages from ChatGPT to GitHub, where a repository workflow applies the changes automatically.

It is designed for a simple loop:

1. Ask ChatGPT to generate either `changed-files.zip` or `openpatch.diff`.
2. Click **Upload to GitHub** next to the generated file on `https://chatgpt.com/`.
3. OpenPatch uploads the file to the target repository.
4. The repository workflow applies the change and commits the real code update to the configured branch.

OpenPatch keeps the browser extension small. It does not parse or merge project code in the browser. GitHub Actions handles the final apply step in a clean Linux environment.

## What It Supports

- Direct upload from ChatGPT attachment buttons when the file can be read from the page.
- Manual upload from the extension popup as a fallback.
- `changed-files.zip` packages with complete changed files.
- Git-generated `.diff` and `.patch` files.
- Repository-relative paths.
- Optional file deletion with `.openpatch-delete.txt` inside a zip package.
- A ChatGPT floating toolbar with two copyable prompts:
  - **ZIP prompt** for `changed-files.zip`
  - **Diff prompt** for `openpatch.diff` generated by Git commands

## Recommended Format

The most reliable format is:

```text
changed-files.zip
├── index.html
├── style.css
└── src/app.js
```

Rules:

- Include only files that are new or changed.
- Store each file at its repository-relative path.
- Do not wrap files in an extra project folder.
- Do not include `.git/`, `.ai-patches/`, `.github/workflows/`, absolute paths, or `../`.
- If files should be removed, add `.openpatch-delete.txt` at the zip root, one repository-relative path per line.

`changed-files.zip` is preferred because it avoids fragile diff context and line-number failures.

## Git Diff Mode

Diff mode is also supported, but ChatGPT should never write diff text by hand. Use the **Diff prompt** so the AI is instructed to modify files first and then run Git:

```bash
git status --short
git add -A
git diff --cached --binary --full-index --find-renames > openpatch.diff
git reset
git apply --check openpatch.diff
```

This produces a standard Git patch and checks it before download.

## Install The Chrome Extension

1. Open Chrome and go to `chrome://extensions`.
2. Enable **Developer mode**.
3. Click **Load unpacked**.
4. Select this project directory.
5. Open the extension options page and configure:
   - GitHub Token
   - Repository, for example `afumu/snake-game-test`
   - Target branch, for example `main` or `master`

The GitHub token should be a fine-grained personal access token with:

- Repository access: the target repository only
- Contents: Read and write
- Metadata: Read

## Install The Target Repository Workflow

Copy this workflow into every repository that should receive OpenPatch uploads:

```text
.github/workflows/apply-ai-patch.yml
```

The template is included in this project at the same path.

The workflow triggers when a new `.zip`, `.diff`, or `.patch` file is added under:

```text
.ai-patches/
```

It applies the upload, commits the resulting code changes, and removes temporary OpenPatch files from the branch.

## How To Use

1. Open ChatGPT at `https://chatgpt.com/`.
2. Use the OpenPatch floating toolbar.
3. Copy either the ZIP prompt or the Diff prompt.
4. Ask ChatGPT to produce the downloadable file.
5. Click **Upload to GitHub** beside the generated file.
6. Wait for the GitHub Actions workflow to commit the applied code changes.

If direct upload cannot read the ChatGPT attachment, use the extension popup and choose the downloaded file manually.

## Repeated Edits

For multiple rounds of AI edits, always generate the next package from the latest project state.

For `changed-files.zip`, every included file is treated as the desired final version of that file. If a zip contains stale file content, it can overwrite newer changes.

For `openpatch.diff`, the patch still depends on the target branch matching the baseline used to generate it. If the branch has moved, the workflow may fail with `patch does not apply`.

# OpenPatch 中文说明

OpenPatch 是一个 Chrome 插件，用来把 ChatGPT 生成的代码修改包发送到 GitHub，然后由仓库里的 GitHub Actions 自动应用这些修改。

它的使用流程很简单：

1. 让 ChatGPT 生成 `changed-files.zip` 或 `openpatch.diff`。
2. 在 `https://chatgpt.com/` 页面点击文件旁边的 **Upload to GitHub**。
3. OpenPatch 把文件提交到目标仓库。
4. 仓库 workflow 自动应用修改，并把真正的代码变更提交到目标分支。

## 推荐格式

最推荐使用：

```text
changed-files.zip
├── index.html
├── style.css
└── src/app.js
```

要求：

- zip 里只放新增或修改后的完整文件。
- 文件路径必须是相对于仓库根目录的路径。
- 不要额外包一层项目目录。
- 不要包含 `.git/`、`.ai-patches/`、`.github/workflows/`、绝对路径或 `../`。
- 如果需要删除文件，在 zip 根目录放 `.openpatch-delete.txt`，每行写一个仓库相对路径。

`changed-files.zip` 比 diff 更稳定，因为它不会因为上下文行号变化而 apply 失败。

## Diff 模式

也支持 `.diff` 和 `.patch`，但不要让大模型手写 diff。请使用插件里的 **Diff prompt**，让 AI 先修改真实文件，再运行 Git 命令生成：

```bash
git status --short
git add -A
git diff --cached --binary --full-index --find-renames > openpatch.diff
git reset
git apply --check openpatch.diff
```

这样生成的是标准 Git patch。

## 安装插件

1. 打开 Chrome 的 `chrome://extensions`。
2. 开启开发者模式。
3. 点击 **Load unpacked**。
4. 选择当前项目目录。
5. 在插件设置页配置：
   - GitHub Token
   - 仓库，例如 `afumu/snake-game-test`
   - 目标分支，例如 `main` 或 `master`

Token 建议使用 fine-grained personal access token，只授权目标仓库，并只给 Contents 读写权限。

## 安装目标仓库 workflow

把下面这个 workflow 放到每个需要接收 OpenPatch 的仓库里：

```text
.github/workflows/apply-ai-patch.yml
```

本项目已经包含这个模板。

workflow 会监听 `.ai-patches/` 下新增的 `.zip`、`.diff` 或 `.patch` 文件，自动应用并提交真实代码变更。

## 多轮修改

如果连续做多轮 AI 修改，每一轮都要基于最新项目状态生成新的包。

对于 `changed-files.zip`，zip 里的文件会被当成最终版本覆盖到仓库中。如果里面是旧内容，就会覆盖掉新修改。

对于 `openpatch.diff`，目标分支必须尽量和生成 diff 时的代码基线一致，否则可能出现 `patch does not apply`。
